# 📊 Revenue-Pulse: Strategic Customer Intelligence Dashboard

An end-to-end analytics pipeline and interactive dashboard built to quantify individual customer contribution and identify high-fidelity revenue growth segments.

## 🔍 Project Overview
In a real-world retail environment, "averages" often hide the truth. I developed **Revenue-Pulse** to solve a critical business challenge: **Identifying exactly who our most valuable customers are and how much revenue they contribute to the platform.** 

By processing over 500,000 messy transactional records, this tool transforms raw data into a **Strategic Revenue Ledger** designed for executive-level decision-making and marketing ROI optimization.

## 🚀 What It Does
The app provides instant, data-backed insights into the health of the customer base:

*   **💰 Individual Customer Attribution:** Tracks and displays "N1 brings X amount" revenue logic for every unique Customer ID.
*   **🟢 Automated Tier Segmentation:** Categorizes users into **High-Value (VIP)**, **Core-Growth**, and **At-Risk** tiers using mathematical quantiles.
*   **📈 Revenue Concentration:** Visualizes the **Pareto Principle (80/20 Rule)** to show which segments drive the most growth.
*   **🚦 Data Integrity Metrics:** Real-time filtering of returns and cancellations ensuring 100% financial accuracy.

## ⚙️ How It Works
I built the application with a focus on **Data Engineering** and **Business Logic**:

*   **🛠️ Professional ETL Pipeline:** Automatically cleans null IDs and filters "C-prefix" (cancellation) invoices to prevent "ghost revenue" reporting.
*   **📊 Statistical Segmentation Engine:** Uses a 90th percentile threshold to identify VIPs and 50th percentile for core users.
*   **💵 Financial Calculation Engine:** Performs real-time Gross Revenue calculation per unique Customer ID.
*   **📈 Interactive Plotly Visuals:** Dynamic Donut and Bar charts allow recruiters to interact with data in real-time.

## 🧰 Tools & Technologies Used
*   **Python** – Core programming and data logic.
*   **Streamlit** – Interactive web dashboard framework.
*   **Pandas/NumPy** – Data engineering and mathematical processing.
*   **Plotly Express** – Advanced interactive data visualizations.
*   **Openpyxl** – High-performance Excel data engine.

## 📂 Dataset Information
The project utilizes the **UCI Online Retail Dataset**, a real-world transactional dataset containing transactions occurring between 01/12/2010 and 09/12/2011.

*   **Data Volume:** 541,909 records.
*   **Business Context:** Includes 4,000+ unique customers across 38 countries.
*   **Cleaning Challenges:** Resolved significant noise including missing Customer IDs, negative quantities (returns), and extreme price outliers.
